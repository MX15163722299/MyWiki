在 Python 中使用 `else` 是为了将没有异常时的代码与可能触发异常的代码分开，从而提高代码的可读性和清晰度。虽然你可以将没有异常的代码直接写在 `try` 块中，但 `else` 块提供了更明确的语义和控制结构，让代码更易于理解和维护。

### 为什么不直接将代码放在 `try` 里面？

假设你将所有代码都放在 `try` 块中，那么 `try` 块会尝试执行所有代码，不管是否可能引发异常。`else` 提供了一种明确的方式来区分哪些代码是没有异常发生时才需要执行的，这样能够：

1. **提高代码的可读性**：
   - `else` 块明确告诉读者：这里是 **没有异常发生时** 执行的代码，这样避免了将正常执行的代码和异常处理代码混在一起。
   - 将处理没有异常的代码放在 `else` 块中，能让程序的逻辑更清晰。
2. **避免在异常处理中执行不必要的代码**：
   - 如果你将代码直接写在 `try` 块中，代码就会被执行，直到发生异常为止。如果你只关心某些代码在没有异常时才运行，放在 `else` 中可以确保这些代码只在没有异常时才被执行。
3. **减少错误和意外情况**：
   - 将没有异常的代码放在 `else` 块中，可以确保你不会错误地在异常发生后继续执行本应在异常未发生时运行的代码。

### 举个例子：

#### 1. **没有 `else` 的情况**

如果你没有使用 `else`，你需要手动判断代码是否会引发异常，可能会导致不必要的复杂性。

```python
try:
    x = int(input("请输入一个数字: "))
    result = 10 / x
    # 如果 x = 0，除法会触发 ZeroDivisionError 异常，下面的代码不会执行
    print(f"结果是: {result}")
except ZeroDivisionError:
    print("除数不能为零!")
except ValueError:
    print("输入无效，请输入一个数字!")
```

#### 2. **有 `else` 的情况**

使用 `else` 可以清晰地分隔成功执行和处理异常的代码块。

```python
try:
    x = int(input("请输入一个数字: "))
    result = 10 / x
except ZeroDivisionError:
    print("除数不能为零!")
except ValueError:
    print("输入无效，请输入一个数字!")
else:
    # 只有在没有异常的情况下才会执行
    print(f"结果是: {result}")
```

### 优点：

- **清晰的逻辑分割**：`else` 明确表示没有异常时应该执行的部分，这种方式结构清晰，易于理解。
- **避免不必要的执行**：异常发生时，不会执行本应该在正常执行时才执行的代码。

### 总结：

`else` 块的主要作用是将没有异常发生时需要执行的代码单独隔离开来，使代码更加清晰、简洁和易于维护。虽然直接写在 `try` 块中也是可以的，但使用 `else` 可以更好地表达代码的意图，避免混乱，并增强代码的可读性。
