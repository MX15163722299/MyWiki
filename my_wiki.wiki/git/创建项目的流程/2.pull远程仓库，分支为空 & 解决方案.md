你在 PyCharm 中新建项目并添加远程仓库（SSH 地址）后，**Pull 时分支列表为空**，通常是由于以下几种原因导致的：

------

## **🔍 可能原因 & 解决方案**

### **1. 远程仓库是空的（没有分支）**

**问题**：

- 如果远程仓库是新创建的，并且还没有提交任何代码，`git pull` 时就不会有可选的分支。
- `git pull` 需要**远程有至少一个分支**，但空仓库没有分支。

✅ **解决方案**：

1. 本地初始化并提交代码

   ：

   ```sh
   git add .
   git commit -m "Initial commit"
   git branch -M master  # 确保分支名称是 master 或 main
   git push -u origin master  # 推送到远程
   ```

2. 然后再尝试 Pull

   ：

   ```sh
   git pull origin master
   ```

------

### **2. 远程仓库存在代码，但本地仓库没有正确关联**

**问题**：

- 你可能已经在远程（Gitee）创建了代码，但本地仓库没有正确拉取远程分支。
- `git remote add origin <SSH地址>` 只是关联了远程仓库，并没有同步分支信息。

✅ **解决方案**：

1. **检查远程仓库是否正确关联**：

   ```sh
   git remote -v
   ```

   你应该看到类似：

   ```
   origin  git@gitee.com:your-username/your-repo.git (fetch)
   origin  git@gitee.com:your-username/your-repo.git (push)
   ```

   如果没有，重新添加：

   ```sh
   git remote add origin git@gitee.com:your-username/your-repo.git
   ```

2. **获取远程分支信息**：

   ```sh
   git fetch origin
   ```

   这会拉取远程的分支信息，但不会修改你的本地代码。

3. **检查远程有哪些分支**：

   ```sh
   git branch -r
   ```

   如果看到 `origin/master` 或 `origin/main`，说明远程有分
